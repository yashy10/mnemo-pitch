01 · The problem

Your coding agent has amnesia.

Every new session starts from zero. So every session, your team re-explains the same things by hand:

  • The stack and conventions — re-pasted into the prompt, again.
  • Which decisions were made, and why — lost the moment the window closes.
  • What's already been tried and failed — so the agent confidently proposes the dead-end you already paid for.

A teammate learns once. An agent forgets every time.

02 · What amnesia costs

It isn't annoying. It's expensive.

demo/payments· production incident
A payment webhook is failing. The agent helpfully adds a retry loop to re-send the charge until it goes through. But the charge already succeeded upstream — so every retry creates a second charge. Customers get billed twice.

This already happened once. It was reverted the same day. But nothing stops the agent from suggesting the exact same fix again tomorrow — it never knew it failed.

The same mistake, on repeat, at machine speed.

03 · Why bolting on "memory" isn't enough

Two ways naive memory makes it worse.

Failure 1 — drift

Stale memory, read as current

Dump transcripts into a vector store and recall resurfaces a decision that's since been reversed. The agent acts on a truth that expired. Confidently wrong is worse than blank.

Failure 2 — exposure

Crown jewels in the cloud

Your code, decisions, and incidents are the most sensitive thing you own. Every memory tool ships them to someone else's servers and asks you to trust it.

The more useful memory gets, the more dangerous it is to send away.

04 · The solution

Mnemo — a local memory plane for your agent.

It sits between your coding agent and the model. It captures what matters, recalls it before every turn, and guards against repeating mistakes — and every byte stays on the box.

Capture

Remember the right things

Typed, attributed knowledge chunks from every session — decisions, dead-ends, conventions. Not raw transcripts.

Recall

Surface them in time

The most load-bearing memory for the repo, prepended to the model before it answers a single turn.

Guard

Stop the repeat

Checks the agent's output against known dead-ends and locked decisions — and blocks the mistake before it ships.

05 · Capture

Typed memory, not transcripts.

Mnemo extracts structured, attributed chunks from each session — each one a small, queryable fact with a type:

decision dead-end finding convention state context
  • Supersession — a better decision retires the old one. Exactly one stays active per topic; history is kept, not confused for the present.
  • Dedup — near-identical chunks are recognised and dropped (0.96 ≥ 0.95). No bloat, no drift.
  • Provenance — every chunk keeps who said it and why. Reasoning, not just text.
  • On the box — extraction runs on a local model. Nothing is sent out to be "remembered."
06 · Recall

The right memory, before every turn.

Before the model sees a turn, Mnemo selects the most load-bearing memory for that repo and prepends it — so the agent walks in already knowing what your team knows.

[mnemo:recall] demo/payments — most load-bearing memory, injected this turn ● dead-end @priya naive retry on a failed webhook double-charged customers in production — reverted same day ● decision @priya retries must send an Idempotency-Key derived from the event id · supersedes the naive resend ● convention @marcus verify the Stripe-Signature header first 3 chunks · selected by relevance + recency · 0 bytes sent off-box
07 · Guard

Block the repeat before it ships.

After the model answers, Guard checks the output against the dead-ends and locked decisions Mnemo already holds. Watch the double-charge loop close:

The mistake that cost you once can't cost you twice.

08 · Why local wins

Local isn't a feature. It's the only architecture that works.

  • Zero egress — inference, memory, and extraction never leave the box. Nothing to leak, nothing to trust.
  • The more agentic Mnemo gets — full-repo context, live execution — the more impossible the cloud becomes. Local stops being a compliance checkbox and becomes the only allowed design.
  • Built for the GB10 — 128 GB unified memory holds the LLM, the embeddings, and the guard model resident at once. A laptop can't; a datacenter you don't control, you won't.

Pull the network cable mid-demo — Mnemo keeps running. Offline · still running.

10 · Live demo

What you'll see, on the box.

  • Capture builds up — typed chunks land in the feed as the agent works the payments repo.
